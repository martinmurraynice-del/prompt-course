第一章

什么是提示词分类？

理解两种不同类型提示词的定义

👤

用户提示词

User Prompt

是您（用户）输入给 AI 的指令、问题或内容。

💡 典型例子：

"帮我写一个Python程序"

🎯 主要作用：

提出具体问题、任务或需求

🔄 变化频率：

每次交互都可能变化

⚙️

系统提示词

System Prompt

是 AI 背后的"隐形指令"，定义了 AI 的基本行为规则、能力范围和交互方式。

💡 典型例子：

"你是一个专业的Python开发工程师..."

🎯 主要作用：

设定角色、规则、输出格式约束

🔄 变化频率：

会话内通常保持不变

第三章

为什么要拆分？

拆分 User Prompt 和 System Prompt 的两个主要原因

1

LLM 的记忆力

每个大语言模型都有固定的上下文窗口（context window），系统提示词会占用一部分 tokens。如果不拆分，重复的系统指令会浪费宝贵的上下文空间。

2

提问效率和质量

系统提示词定义好 AI 的角色和规则后，后续提问只需要输入具体的 User Prompt，指令更清晰，AI 理解更准确，输出质量更高。

上下文长度对比（K tokens）

💡 小贴士： DeepSeek-R1 支持 128K tokens (≈ 6万字汉字)

类比： 上下文长度就是 AI 的"工作记忆力"，上下文不足模型会"断片"

"上下文不够？模型会'断片'，前文忘后文。"

第五章

正向提问，避免负面词语

❌

错误写法

不要不像人一样

不要太专业

❌ 问题：使用负面词语，AI 可能误解你的真实需求

要口语化，像和闺蜜聊天

字数、格式、风格都可以限定

👥

第六章

ICL 技术：给模型少量样本

In-Context Learning：把样本放到盒子里让模型学习

0️⃣

Zero-Shot

零样本提示

示例数量：

0 个示例

提示：

"翻译：Hello World"

直接提问，不给任何示例

1️⃣

One-Shot

单样本提示

示例数量：

1 个示例

提示：

狗 → 动物

猫 → ?

给1个示例，让AI学习模式

🔢

Few-Shot

少样本提示

示例数量：

2-5 个示例

提示：

狗 → 动物

猫 → 动物

鱼 → 动物

鸟 → ?

给多个示例，让AI更准确地学习

💡 ICL 的核心思想

📦 什么是"盒子"？

在提示词中，我们把示例放在 System Prompt 或 User Prompt 的开头，形成一个"上下文盒子"。模型通过阅读盒子里的样本，学习我们期望的输出模式。

🎯 什么时候用？

  • • 需要特定格式输出时
  • • 任务比较抽象难描述时
  • • 想要保持输出风格一致时
  • • 处理分类、转换类任务时
第七章

Prompt 到底是什么？

= 表达。人类向 AI 表达

乔哈里视窗模型

Johari Window 是一种关于沟通的技巧和理论，由乔瑟夫和哈里提出。视窗理论将人际沟通的信息比作一个个窗子，它被分为4个区域。

🟢 公开象限

自己知道，他人也知道。这是最有效的沟通区域。在 AI 提示中，这是我们明确表达给 AI 的信息。

🔴 隐私象限

自己知道，他人不知道。我们需要通过提示词，把 AI 不知道的信息告诉它。

🟠 盲点象限

自己不知道，他人知道。AI 可能发现我们的盲点，通过反馈帮助我们成长。

🟣 潜能象限

自己不知道，他人也不知道。通过与 AI 协作，可以探索未知的潜能。

第八章

人与 AI 的协作四象限

基于乔哈里视窗，理解人与 AI 的协作模式

🔵 学习象限

我不知道，AI知道

这是向 AI 学习的最佳场景。通过正确提问，让 AI 传授我们不懂的知识。关键是学会如何提出好问题。

🩷 助理象限

我知道，AI知道

这是最常见的协作模式。我们明确知道要什么，AI 也具备相应能力。关键是下达清晰的指令。

🟠 共创象限

我不知道，AI不知道

这是最具创造力的区域。双方都不知道答案，通过协作探索新的可能性。适合头脑风暴和创新。

🟢 投喂象限

我知道，AI不知道

这是知识传递的场景。我们需要把 AI 不知道的信息（如私有数据、最新信息）通过提示词喂给它。

📚

总结

💬 User Prompt

用户每次输入的问题、需求

⚙️ System Prompt

定义 AI 角色、规则、行为

拆分原因：

1️⃣ 节约 LLM 上下文窗口 → 提高记忆力利用率

2️⃣ 提高提问效率和输出质量 → 指令更清晰

核心技能：

🎯 ICL 技术：Zero/One/Few-Shot 提示

🪟 乔哈里视窗：理解信息传递

🤝 四象限协作：学习/助理/共创/投喂

会提问比会做题更重要
